# 3. 시료 주문 (접수)

## 개요
고객의 시료 요청을 주문 담당자가 시스템에 등록한다.
주문 생성 시 상태는 항상 `RESERVED`로 설정된다.

---

## 입력값

| 필드 | 타입 | 제약 조건 |
|------|------|-----------|
| 시료 ID | String | 시스템에 등록된 시료만 허용 |
| 고객명 | String | 공백 불가 |
| 주문 수량 | int | 1 이상 정수 |

---

## 처리 흐름

```
시료 ID 입력
    │
    ▼
시료 존재 여부 확인
    ├─ 없음 → "등록되지 않은 시료 ID입니다." 출력 후 재입력
    └─ 있음
        │
        ▼
    고객명 입력
        │
        ▼
    주문 수량 입력
        │
        ▼
    입력 내용 확인 화면 출력
    (시료명, 고객명, 수량)
        │
    [Y] 예약 접수
        │
        ▼
    주문번호 자동 생성
        │
        ▼
    Order 저장 (상태: RESERVED)
        │
        ▼
    "예약 접수 완료" + 주문번호 + 현재 상태 출력
    [N] 취소 → 메인 메뉴로 복귀
```

---

## 주문번호 생성 규칙

```
형식: ORD-{YYYYMMDD}-{순번 4자리}
예시: ORD-20260612-0001

- 날짜별 독립 순번. 날짜 변경 시 0001 리셋
- 재실행 후 같은 날의 순번은 data/orders.json 에서 이어받아 유지
```

---

## 출력 예시

```
[2] 시료 주문

시료 ID   > S-003
고객명    > 삼성전자 파운드리
주문 수량 > 200

입력 내용 확인
  시료   : SiC 파워기판-6인치 (S-003)
  고객   : 삼성전자 파운드리
  수량   : 200 ea

[Y] 예약 접수   [N] 취소
선택 > Y

예약 접수 완료.
  주문번호  : ORD-20260612-0001
  현재 상태 : RESERVED

※ 재고 확인 및 승인은 [3] 주문 승인/거절 메뉴에서 진행하세요.
```

---

## 데이터 모델

```cpp
class Order {
    std::string   orderId;       // 주문번호 (ORD-YYYYMMDD-NNNN)
    std::string   sampleId;      // 시료 ID (FK)
    std::string   customerName;  // 고객명
    int           quantity;      // 주문 수량
    OrderStatus   status;        // 주문 상태
    std::string   createdAt;     // 주문 생성일시
    std::string   updatedAt;     // 상태 변경일시
};
```

---

## 영속성
- 저장 경로: `data/orders.json`
- 주문 생성 즉시 `persist()` 호출

---

## 구현 클래스
- `Order` — 데이터 모델
- `OrderStatus` — 상태 Enum (RESERVED / REJECTED / PRODUCING / CONFIRMED / RELEASED)
- `OrderRepository` — CRUD + JSON 영속성, 순번 관리
- `OrderController` — 주문 접수 흐름 제어
- `OrderView` — 입력 수집 및 결과 출력
