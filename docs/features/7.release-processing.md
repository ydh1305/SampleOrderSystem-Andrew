# 7. 출고 처리

## 개요
`CONFIRMED` 상태의 주문에 대해 출고를 실행한다.
출고 처리 후 주문 상태는 `RELEASED`로 전환된다.

---

## 화면 구성

- 출고 가능한 주문 목록 (`CONFIRMED` 상태만 표시)
- 출고할 주문 선택 → 처리 결과 출력

---

## 처리 흐름

```
CONFIRMED 주문 목록 출력
    │
    ├─ 목록이 없음 → "출고 가능한 주문이 없습니다." 출력 후 복귀
    └─ 목록이 있음
        │
        ▼
    출고할 번호 입력
        │
        ▼
    선택된 주문 상세 확인
        │
    [Y] 출고 확정
        │
        ▼
    주문 상태: CONFIRMED → RELEASED
    처리 일시 기록
        │
        ▼
    출고 완료 정보 출력
    [N] 취소 → 목록으로 복귀
```

---

## 출력 예시

```
[6] 출고 처리

출고 가능 주문 (CONFIRMED)
번호  주문번호          고객         시료                수량
[1]   ORD-20260612-0042  SK하이닉스   실리콘 웨이퍼-8인치  150 ea
[2]   ORD-20260612-0035  DB하이텍     포토레지스트-PR7     400 ea

출고할 번호 > 1

출고 처리 완료.
  주문번호  : ORD-20260612-0042
  출고 수량 : 150 ea
  처리 일시 : 2026-06-12 09:34:02
  상태      : CONFIRMED → RELEASED
```

---

## 입력 유효성
- 목록에 표시된 번호만 허용
- 범위 외 입력 시 "잘못된 입력입니다." 출력 후 재입력

---

## 상태 전이

```
CONFIRMED → RELEASED
```

- `CONFIRMED` 이외의 상태 주문은 출고 목록에 표시되지 않음
- 출고 처리 시 재고 추가 차감 없음 (재고는 승인 시점에 이미 차감됨)

---

## 영속성
- `data/orders.json` — 주문 상태 RELEASED 업데이트, `updatedAt` 기록

---

## 구현 클래스
- `ReleaseController` — 출고 목록 조회 및 처리 흐름 제어
- `ReleaseView` — CONFIRMED 목록 출력 및 처리 결과 출력
- `OrderRepository` — 주문 상태 업데이트
