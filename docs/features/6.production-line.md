# 6. 생산라인

## 개요
재고 부족으로 `PRODUCING` 상태가 된 주문에 대해 생산라인이 가동된다.
단일 라인으로 FIFO 방식으로 처리하며, 생산 완료 시 주문 상태를 `CONFIRMED`로 전환한다.

---

## 화면 구성

### 생산라인 현황 화면
1. **현재 처리 중** — 현재 가동 중인 생산 작업 상세
2. **대기 중인 주문** — 생산 큐에 대기 중인 작업 목록 (FIFO 순)

---

## 생산량 계산 공식

```
부족분      = 주문 수량 - 현재 재고
실 생산량   = ceil(부족분 / (수율 × 0.9))   ← 수율 오차 10% 감안
총 생산시간 = 평균 생산시간(min/ea) × 실 생산량
```

**예시**: 부족분 170ea, 수율 0.92, 평균 생산시간 0.8 min/ea
```
실 생산량   = ceil(170 / (0.92 × 0.9)) = ceil(205.3) = 206 ea
총 생산시간 = 0.8 × 206 = 164.8 min ≈ 165 min
```

---

## 생산 완료 처리 흐름

```
생산 완료 이벤트 (수동 진행 처리)
    │
    ▼
생산된 정상 시료 수 계산
  정상품 수 = floor(실 생산량 × 수율)
    │
    ▼
재고 증가: stock += 정상품 수
    │
    ▼
주문 상태: PRODUCING → CONFIRMED
    │
    ▼
ProductionJob 상태: RUNNING → DONE
    │
    ▼
큐에 다음 대기 작업이 있으면 자동 시작
```

---

## 스케줄링 전략

- **FIFO (First In, First Out)**: 주문 승인 순서대로 생산
- **단일 라인**: 한 번에 하나의 작업만 처리
- 현재 작업이 완료되면 큐의 첫 번째 대기 작업이 자동으로 시작

---

## 생산 진행 처리 (수동)

생산은 실제 설비 연동 없이 **담당자가 생산 완료를 수동으로 입력**하는 방식으로 처리한다.

```
현재 처리 중인 작업 확인
    │
    ▼
[C] 생산 완료 처리 입력
    │
    ▼
재고 증가 + 주문 CONFIRMED 전환 + 다음 작업 시작
```

## 데이터 모델

```cpp
enum class JobStatus { WAITING, RUNNING, DONE };

class ProductionJob {
    std::string jobId;
    std::string orderId;
    std::string sampleId;
    int         shortage;      // 부족분
    int         actualProd;    // 실 생산량
    double      totalTime;     // 총 생산시간 (min)
    JobStatus   status;        // WAITING / RUNNING / DONE
    std::string enqueuedAt;    // 큐 등록 시각
    std::string startedAt;     // 생산 시작 시각
    std::string completedAt;   // 생산 완료 시각
};
```
