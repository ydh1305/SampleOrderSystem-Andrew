# 2. 시료 관리

## 개요
시료(Sample)는 시스템의 가장 기본 단위. 시스템에 등록된 시료만 주문 가능하다.
등록, 전체 조회, 검색 세 가지 기능을 제공한다.

---

## 서브 메뉴

| 번호 | 기능 |
|------|------|
| 1 | 시료 등록 |
| 2 | 시료 목록 조회 |
| 3 | 시료 검색 |
| 0 | 뒤로 (메인 메뉴) |

---

## 2-1. 시료 등록

### 입력값
| 필드 | 타입 | 제약 조건 |
|------|------|-----------|
| 시료 ID | String | 고유값, 공백 불가, 중복 불가 (예: `S-001`) |
| 시료명 | String | 공백 불가 |
| 평균 생산시간 | double | 양수, 단위: min/ea |
| 수율 | double | 0.0 초과 ~ 1.0 이하 |
| 초기 재고 | int | 0 이상 정수 |

### 처리 흐름
```
입력값 수집
    │
    ▼
유효성 검사
    ├─ 실패 → 오류 메시지 출력 후 재입력
    └─ 성공
        │
        ▼
    입력 내용 확인 화면 출력
    [Y] 등록 확정 → Repository 저장 → "등록 완료" 출력
    [N] 취소      → 서브 메뉴로 복귀
```

### 유효성 검사 항목
- 시료 ID 중복 여부 (기존 데이터와 비교)
- 수율 범위: `0.0 < yieldRate <= 1.0`
- 평균 생산시간: `avgProductionTime > 0`
- 초기 재고: `stock >= 0`

---

## 2-2. 시료 목록 조회

### 출력 컬럼
| 컬럼 | 설명 |
|------|------|
| ID | 시료 ID |
| 시료명 | 시료 이름 |
| 평균 생산시간 | min/ea 단위 |
| 수율 | 소수점 2자리 |
| 현재 재고 | ea 단위 |

### 처리 흐름
- Repository에서 전체 목록 조회 후 테이블 형식으로 출력
- 등록된 시료가 없으면 "등록된 시료가 없습니다." 출력

---

## 2-3. 시료 검색

### 검색 기준
- 시료명 부분 일치 검색 (대소문자 구분 없음)

### 처리 흐름
```
검색어 입력
    │
    ▼
이름에 검색어가 포함된 시료 목록 출력
    └─ 결과 없음 → "검색 결과가 없습니다." 출력
```

### 출력 형식
- 조회와 동일한 테이블 형식

---

## 데이터 모델

```cpp
class Sample {
    std::string sampleId;        // 고유 ID
    std::string name;            // 시료명
    double      avgProductionTime; // 평균 생산시간 (min/ea)
    double      yieldRate;       // 수율 (0.0 ~ 1.0)
    int         stock;           // 현재 재고
};
```

---

## 영속성
- 저장 경로: `data/samples.json`
- 등록·재고 변경 시 즉시 `persist()` 호출

---

## 구현 클래스
- `Sample` — 데이터 모델
- `SampleRepository` — CRUD + JSON 영속성
- `SampleController` — 등록·조회·검색 흐름 제어
- `SampleView` — 입력 수집, 목록·검색 결과 출력
