# PRD - 반도체 시료 생산주문관리 시스템

## 1. 배경 및 목적

가상의 반도체 회사 **S-Semi**는 다양한 종류의 반도체 시료(Sample)를 생산하여 연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 납품한다. 시료는 주문 접수 후 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 고객에게 출고된다.

주문량 급증으로 인해 엑셀·메모장 기반 관리에서 다음 문제가 발생했다:
- 주문 처리 여부 불명확
- 공정 완료 시점 파악 불가
- 재고가 충분함에도 불필요한 생산 가동

이를 해결하기 위해 **콘솔 기반 반도체 시료 생산주문관리 시스템**을 개발한다.

---

## 2. 사용자 및 역할

| 역할 | 설명 |
|------|------|
| 고객 | 시료 요청자. 이메일로 시료 요청 |
| 주문 담당자 | 고객 요청을 시스템에 주문서로 등록·관리 |
| 생산 담당자 | 개발 시료 등록, 주문 수신 후 승인 또는 거절 |

**흐름**: 고객 → (시료 요청) → 주문 담당자 → (주문서 전달) → 생산 담당자 → (승인/거절) → 주문 담당자

---

## 3. 주문 상태 정의

| 상태 | 의미 |
|------|------|
| RESERVED | 주문 접수 |
| REJECTED | 주문 거절 (모니터링 제외) |
| PRODUCING | 주문 승인 완료 및 재고 부족으로 생산 중 |
| CONFIRMED | 주문 승인 완료 및 출고 대기 중 |
| RELEASED | 출고 완료 |

> REJECTED는 정상 흐름 외의 상태이며 모니터링에서 제외된다.

---

## 4. 기능 요구사항

### 4.1 메인 메뉴

- 기능별 선택 화면 표시
- 전체 시료에 대한 요약 정보 표시 (등록 시료 수, 총 재고, 전체 주문 수, 생산라인 대기 수)

| 메뉴 번호 | 메뉴명 | 설명 |
|-----------|--------|------|
| 1 | 시료 관리 | 새로운 시료 등록, 목록 조회, 이름 검색 |
| 2 | 시료 주문 | 고객 주문 접수 (RESERVED 생성) |
| 3 | 주문 승인/거절 | RESERVED 주문 승인 또는 거절 처리 |
| 4 | 모니터링 | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 5 | 생산라인 조회 | 현재 생산 중인 시료 및 대기 중인 생산 큐 확인 |
| 6 | 출고 처리 | CONFIRMED 상태 주문에 대해 출고 실행 |
| 0 | 종료 | 시스템 종료 |

---

### 4.2 시료 관리

시료(Sample)는 시스템의 가장 기본 단위. 시스템에 등록된 시료만 주문 가능.

**시료 속성**
- 시료 ID (고유값)
- 이름
- 평균 생산시간 (min/ea)
- 수율 = 정상 시료 수 / 총 생산 시료 수 (예: 0.92)

**세부 기능**

| 기능 | 설명 |
|------|------|
| 시료 등록 | 새로운 시료를 시스템에 추가 |
| 시료 조회 | 등록된 모든 시료 목록 확인 (현재 재고 수량 포함) |
| 시료 검색 | 이름 등 속성으로 특정 시료 검색 |

---

### 4.3 시료 주문 (접수)

고객이 시료를 요청하면 주문 담당자가 주문을 생성한다.

**예약 시 입력값**
- 시료 ID
- 고객명
- 주문 수량

**처리 결과**
- 주문 상태: RESERVED
- 주문번호 자동 생성 (예: `ORD-YYYYMMDD-NNNN`)

---

### 4.4 주문 승인/거절

RESERVED 상태의 주문 목록을 확인하고 특정 주문에 대해 승인 또는 거절을 처리한다.

**주문 승인**
- 재고 확인 후 2가지 경로로 자동 처리
  - 재고 충분 → 즉시 CONFIRMED 전환
  - 재고 부족 → 생산라인 자동 등록 + 주문 상태 PRODUCING 전환

**주문 거절**
- 즉시 REJECTED 상태로 전환

---

### 4.5 모니터링

담당자가 현재 시스템 상태를 한눈에 파악할 수 있도록 구성.

**주문량 확인**
- 상태별 주문 수 표시: RESERVED / CONFIRMED / PRODUCING / RELEASED
- REJECTED는 표시 제외

**재고량 확인**
- 각 시료별 현재 재고 수량 표시
- 재고 상태 표기

| 상태 | 기준 |
|------|------|
| 여유 | 주문 대비 재고 충분 |
| 부족 | 주문 대비 재고 수량 부족 |
| 고갈 | 수량이 0인 상태 |

---

### 4.6 생산라인

재고 부족으로 PRODUCING 상태가 된 주문에 대해 생산라인이 가동된다.

**생산 수량 계산**
- 부족분 = 주문 수량 - 현재 재고
- 실 생산량 = `ceil(부족분 / (수율 × 0.9))` ← 수율 오차 10% 감안
- 총 생산 시간 = 평균 생산시간 × 실 생산량

**생산 완료 처리**
- 생산 완료 시 주문 상태: PRODUCING → CONFIRMED

**생산 현황 표기**
- 현재 생산 중인 시료 정보 표시 (주문 정보, 현재까지의 생산량 등)

**대기 주문 확인**
- 생산 큐(FIFO 방식)로 대기 중인 주문 목록 출력
- 스케줄링 전략: **FIFO (선입선출)**

**생산라인 특성**
- 하나의 생산라인은 시료를 하나씩 생산
- 주문이 들어온 시료에 대해서만 생산

---

### 4.7 출고 처리

CONFIRMED 상태의 주문에 대해 출고를 실행한다.

- 출고 가능 주문 목록(CONFIRMED) 표시
- 특정 주문 선택 후 출고 실행
- 처리 후 주문 상태: CONFIRMED → RELEASED

---

## 5. 비기능 요구사항

- 시스템은 **콘솔 기반**으로 동작
- 담당자가 직접 명령을 입력하여 시료를 등록하고 주문을 처리
- **데이터 영속성**: 애플리케이션 재실행 후에도 데이터 유지
- 화면 구성은 자유롭게 결정 가능 (예시 UI는 참고용)
